Reading Momentum

Visual Design Language

Version 1.0

This document does not describe colors or components.

It describes the feeling every screen should create.

Every future design should be compared against these principles.

⸻

First Impression

When someone opens Reading Momentum for the first time, they should not think:

“This is another productivity app.”

They should think:

“This feels peaceful.”

The application should immediately slow the user down.

The experience should feel similar to opening a hardcover book.

There should never be visual urgency.

Nothing should compete for attention.

The application should quietly guide the user toward reading.

⸻

Design Philosophy

Reading Momentum should feel like:

Walking into a quiet independent bookstore.

Soft music playing.

Warm lighting.

Wood shelves.

Coffee nearby.

Comfortable leather chair.

Books displayed intentionally.

People quietly discussing books.

Nothing feels rushed.

Nothing is trying to grab attention.

That emotional experience should be translated into software.

⸻

Visual Hierarchy

Everything on screen has a purpose.

Only one thing should ever demand attention.

Current reading should always be the primary focus.

Everything else supports it.

When opening the app the eye should naturally move in this order:

Greeting

↓

Current Book

↓

Reading Progress

↓

Reflection

↓

Friends

↓

Book of the Month

Nothing else should interrupt that flow.

⸻

Information Density

The interface should never feel crowded.

Large margins.

Large spacing.

Few components.

Each screen should answer one question.

Home

“What should I read?”

Library

“What books am I reading?”

Community

“What conversations should I join?”

Profile

“How is my reading journey going?”

Do not combine multiple purposes onto one screen.

⸻

Component Philosophy

Cards should feel like physical objects.

Almost like note cards placed on a wooden desk.

Soft shadows.

Rounded corners.

Comfortable spacing.

Minimal borders.

No heavy outlines.

Cards should feel tactile.

⸻

Motion

Nothing should pop.

Nothing should bounce.

Nothing should flash.

Everything should glide.

Examples:

Cards gently appear.

Book covers slightly expand.

Progress bars slowly fill.

Pages slide naturally.

Glass cards softly fade.

Animation should never distract.

Animation should reinforce calmness.

⸻

Reading Experience

Reading is sacred.

The application should respect it.

Whenever possible:

Reduce interface.

Increase typography.

Increase whitespace.

Increase focus.

The application should encourage users to leave the app and continue reading.

⸻

Book Covers

Book covers are artwork.

Treat them that way.

Large.

Centered.

Never cropped aggressively.

Never tiny thumbnails.

Soft shadows.

Slight elevation.

The eye should naturally be drawn toward the current book.

⸻

Typography

Typography is the second most important visual element after books.

Headings should feel editorial.

Paragraphs should resemble magazine layouts.

Nothing should feel compressed.

Text should invite reading.

⸻

White Space

Whitespace is not empty space.

Whitespace creates relaxation.

If a designer asks:

“Should we add another card?”

The answer is probably no.

Negative space is part of the interface.

⸻

Community

Community should not resemble social media.

Discussions should resemble conversations inside a bookstore.

Replies should resemble notes in a book.

No giant reaction counts.

No popularity contests.

Quality over quantity.

⸻

Reviews

Reviews should resemble beautifully designed essays.

Large typography.

Comfortable reading width.

Elegant pull quotes.

Minimal interface.

Reading should always be more important than interaction.

⸻

Progress

Progress should never create anxiety.

Progress indicators should encourage.

Not pressure.

Reading progress bars should be thin.

Momentum should be subtle.

Achievements should celebrate quietly.

⸻

Glassmorphism

Glass is reserved for moments worth remembering.

Book finished.

Achievement unlocked.

One year streak.

Book of the Month winner.

Never use glass simply because it looks modern.

It should create emotional contrast.

⸻

Color Psychology

Warm colors communicate comfort.

Forest green communicates growth.

Muted gold communicates achievement.

Cream backgrounds resemble paper.

Dark mode resembles walnut bookshelves.

Everything should feel organic.

Never synthetic.

⸻

Navigation

Navigation should disappear.

Users should never think about navigation.

Four tabs.

Nothing more.

Every interaction should require as few decisions as possible.

⸻

Photography

If imagery is used:

Books.

Coffee.

Wood.

Paper.

Libraries.

Soft light.

Nothing staged.

Nothing corporate.

⸻

Sound

If sound is ever introduced:

Soft page turn.

Gentle bookmark.

Muted celebration.

Nothing loud.

Nothing arcade-like.

⸻

Emotional Check

Every screen should pass this test.

Does this make reading feel more relaxing?

Does this make the user want to pick up their book?

Does this reduce cognitive load?

Does this feel timeless?

Would Apple ship this?

Would this still feel beautiful in five years?

If the answer to any of these questions is no, redesign the screen.

⸻

Design Rule

Reading Momentum should never feel like software.

It should feel like a place people want to return to.

The interface should disappear.

The books should become the experience.
